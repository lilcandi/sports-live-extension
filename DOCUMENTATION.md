## 1. 项目概述

**赛事实时播报** 是一款轻量级 Chrome 浏览器扩展，实时播报体育赛事、电竞比分和中文体育资讯。

### 核心特性

- 🏆 **60+ 赛事覆盖** — 足球/篮球/赛车/网球/乒乓球/电竞六大类
- 📰 **13 个资讯源** — 虎扑/知乎/微博/懂球帝/直播吧等中文资讯
- 🔥 **热度排名系统** — S+/S/A/B/C 五级热度，动态计算
- ⚙️ **30+ 设置选项** — 高度可定制的通知、显示、资讯、数据管理
- 🔔 **四级通知** — Badge → Popup → 系统通知 → 网页浮层
- 💾 **数据备份** — 一键导出/导入设置和数据

### 项目定位

- **轻量**：纯原生 JS，无框架依赖，占用资源少
- **免费**：所有数据源均为免费，无需付费
- **中文**：全中文界面和中文资讯源
- **低打扰**：可配置热度门槛，只推送重要赛事

## 2. 技术栈

### 2.1 核心技术

| 技术 | 版本/规格 | 用途 |
|------|-----------|------|
| Manifest V3 | Chrome 88+ | 扩展标准 |
| Service Worker | - | 后台数据拉取 |
| JavaScript | ES6+ | 核心逻辑 |
| HTML5 | - | 弹窗和设置页面 |
| CSS3 | - | 样式和动画 |
| Chrome Extensions API | MV3 | 扩展能力 |

### 2.2 Chrome API 使用

| API | 用途 | 权限 |
|-----|------|------|
| `chrome.storage` | 本地存储 + 同步设置 | `storage` |
| `chrome.alarms` | 定时刷新数据 | `alarms` |
| `chrome.notifications` | 系统通知 | `notifications` |
| `chrome.tts` | 语音播报 | `tts` |
| `chrome.action` | 扩展图标和角标 | - |
| `chrome.runtime` | 消息通信 | - |
| `chrome.tabs` | 打开新标签页 | - |
| `chrome.contentScripts` | 网页浮层 | 内容脚本 |

### 2.3 数据解析技术

| 解析方式 | 适用场景 | 优点 | 缺点 |
|----------|----------|------|------|
| 正则表达式 | HTML 页面 | 轻量、快速、不易被检测 | 结构变化时需更新 |
| JSON.parse | API 接口 | 结构化、准确 | 依赖 API 可用性 |
| XML 正则 | RSS 源 | 标准格式 | RSSHub 不稳定 |

### 2.4 数据存储

| 存储类型 | 容量 | 用途 | 同步 |
|----------|------|------|------|
| `chrome.storage.local` | ~5MB | 赛事缓存、资讯缓存、历史记录 | ❌ 本地 |
| `chrome.storage.sync` | ~100KB | 用户设置、收藏联赛 | ✅ 跨设备 |

## 3. 安装使用

### 3.1 环境要求

- **浏览器**：Chrome 88+ / Edge 88+
- **操作系统**：Windows / macOS / Linux / ChromeOS
- **网络**：可访问 500.com、虎扑等体育网站

### 3.2 安装方法

#### 方式一：开发者模式加载（推荐）

1. 下载或克隆本仓库
   ```bash
   git clone https://github.com/lilcandi/sports-live-extension.git
   ```

2. 打开扩展管理页面
   - Chrome: 地址栏输入 `chrome://extensions/`
   - Edge: 地址栏输入 `edge://extensions/`

3. 开启开发者模式
   - Chrome: 右上角「开发者模式」开关
   - Edge: 左侧边栏「开发人员模式」开关

4. 加载扩展
   - 点击「加载已解压的扩展程序」
   - 选择项目根目录（包含 manifest.json 的文件夹）

5. 完成
   - 扩展图标出现在工具栏
   - 点击图标即可使用

#### 方式二：从应用商店安装

> 暂未上架 Chrome 应用商店，敬请期待。

### 3.3 基本使用

#### 查看赛事

1. 点击工具栏扩展图标
2. 默认显示「热度排名」页面
3. 点击顶部 Tab 切换不同页面
4. 点击任意赛事/资讯卡片，在新标签页打开详情

#### 调整设置

1. 点击弹窗右上角 ⚙️ 图标
2. 或右键扩展图标 → 选项
3. 在设置页面调整各项配置
4. 点击「保存设置」生效

#### 收藏联赛

1. 打开设置页面
2. 在「关注联赛」区域勾选感兴趣的联赛
3. 保存设置

### 4.3 七个功能页面

#### Tab 1：🔥 热度排名（默认首页）

**功能**：
- 显示所有赛事按热度从高到低排序
- TOP3 显示金/银/铜牌徽章（🥇🥈🥉）
- 热度条颜色随热度变化（红>黄>蓝>绿）
- 支持按运动分类筛选
- 热度分 = 基础关注度 + 实时状态加成 + 比分激烈度

**热度计算规则**：
```
热度 = 基础关注度分
     + 状态加成（进行中+30，即将开始+15，已结束+10）
     + 比分加成（总得分≥5 +10，≥3 +5；分差≤1 +10，≤3 +5）
最高上限：100分
```

#### Tab 2：⚽ 实时赛事

**功能**：
- 显示所有正在进行和即将开始的赛事
- 按热度从高到低排序
- 进行中比赛标红显示，LIVE 标识闪烁
- 支持按运动分类筛选
- 点击卡片跳转详情页

#### Tab 3：📰 中文资讯

**功能**：
- 聚合所有启用资讯源的内容
- 按热度排序
- 支持按来源筛选
- 点击卡片跳转原文
- 自动去重（标题+来源）

#### Tab 4：📅 未来赛程

**功能**：
- 显示未来60天内的赛事
- 电竞赛事显示详细赛程信息
- 支持按运动分类筛选
- 按热度排序

#### Tab 5：🔍 历史搜索

**功能**：
- 搜索一年内的历史赛果
- 支持关键词搜索（队伍/联赛/赛事名）
- 支持时间范围筛选（7天/30天/90天/180天/1年）
- 按热度排序显示结果
- 一键清空历史

#### Tab 6：⭐ 我的收藏

**功能**：
- 显示已收藏的联赛列表
- 显示联赛级别和热度值
- 支持取消收藏

#### Tab 7：📊 昨日总结

**功能**：
- 每日早8点自动生成
- 统计昨日比赛数量和分类
- 高光比赛（高得分/胶着比赛）
- 联赛统计排行

### 4.4 通知系统

四级通知机制（由轻到重）：

1. **Badge 角标**
   - 扩展图标右上角数字
   - 显示更新的比赛数量
   - 绿色背景
   - 不打扰用户

2. **Popup 静默更新**
   - 打开弹窗时看到最新数据
   - 完全静默，不打扰

3. **系统通知**
   - 高热度比赛结束时弹出
   - 可设置最低热度门槛
   - 可选择是否播放声音
   - 点击通知可跳转详情

4. **网页浮层**
   - 浏览任意网页时右下角弹出
   - 比分更新 + 资讯更新
   - 点击跳转详情页
   - 自动消失（5-6秒）
   - 滑动进入动画

### 4.5 语音播报

**支持的语音类型**：
- 进球/得分语音播报
- 比赛结束语音播报

**语音设置**：
- 支持4档语速：慢速(0.7x)/正常(0.85x)/稍快(1.0x)/快速(1.2x)
- 使用 Chrome 浏览器内置 TTS 引擎
- 中文语音（zh-CN）

### 4.6 数据管理

**数据统计**：
- 赛果记录数
- 资讯缓存数
- 收藏联赛数

**数据操作**：
- 导出数据（JSON格式，包含sync和local）
- 导入数据（JSON格式，覆盖现有数据）
- 清空全部数据
- 自动清理历史（可配置保留天数：30/90/180/365天/永久）

## 5. 不可用/受限功能

### 5.1 不可用功能

| 功能 | 状态 | 原因 | 预计修复 |
|------|------|------|----------|
| 电竞实时比分 | ❌ 不可用 | Liquipedia/HLTV/vlr.gg 被 Cloudflare 反爬 | 暂无方案 |
| 乒乓球实时比分 | ❌ 不可用 | 无合适的免费数据源 | 暂无方案 |
| 推送通知（Push） | ❌ 不可用 | MV3 限制，无法主动推送 | - |
| 多设备同步赛事数据 | ❌ 不可用 | sync storage 容量限制 | - |
| 移动端支持 | ❌ 不可用 | 移动端 Chrome 不支持扩展 | - |
| 暗黑/浅色主题切换 | ❌ 不可用 | 暂未实现 | 后续版本 |
| 多语言支持 | ❌ 不可用 | 仅中文 | 后续版本 |
| Firefox 支持 | ❌ 不可用 | API 差异 | 暂无计划 |
| Safari 支持 | ❌ 不可用 | 扩展体系不同 | 暂无计划 |

### 5.2 受限功能

| 功能 | 限制程度 | 说明 |
|------|----------|------|
| RSSHub 资讯源 | ⚠️ 不稳定 | 公共 RSSHub 实例可能超时或被封，已配置备用源 fallback |
| 500.com 赛果 | ⚠️ 仅竞彩赛事 | 只能获取有竞彩开售的比赛 |
| TheSportsDB | ⚠️ 数据延迟 | 免费 API 有速率限制，数据可能延迟 |
| 虎扑爬取 | ⚠️ 可能失效 | 网站结构变化可能导致解析失败 |
| 懂球帝爬取 | ⚠️ 可能失效 | 网站结构变化可能导致解析失败 |
| 直播吧爬取 | ⚠️ 可能失效 | 网站结构变化可能导致解析失败 |
| 新浪体育爬取 | ⚠️ 可能失效 | 网站结构变化可能导致解析失败 |
| 腾讯体育爬取 | ⚠️ 可能失效 | 网站结构变化可能导致解析失败 |
| 通知频率 | ⚠️ 系统限制 | Chrome 通知可能被系统静音或归类 |
| 语音播报 | ⚠️ 依赖系统 TTS | 需要系统安装中文语音包 |
| 后台刷新 | ⚠️ 可能延迟 | Service Worker 可能被浏览器挂起 |
| 数据存储 | ⚠️ 容量限制 | local storage 约 5MB 上限 |

### 5.3 已知限制

1. **数据刷新频率限制**
   - 赛果数据：最快 30 秒（默认 2 分钟）
   - 资讯数据：最快 5 分钟（默认 15 分钟）
   - 受 Chrome Alarms API 最小 1 分钟限制

2. **存储限制**
   - sync storage: 100KB（设置同步）
   - local storage: 5MB（本地缓存）
   - 历史数据自动清理

3. **后台运行限制**
   - Service Worker 可能被浏览器挂起
   - 非活动状态下可能延迟刷新

4. **内容安全策略**
   - 无法访问需要登录的内容
   - 无法绕过付费墙内容

## 8. 已知问题与限制

### 8.1 数据相关

1. **电竞赛事无实时比分**
   - 现象：电竞页面只有赛程信息，没有实时比分
   - 原因：主流电竞数据站被 Cloudflare 保护
   - 解决方案：使用静态赛程数据
   - 预计修复：暂无完美解决方案

2. **500.com 仅覆盖竞彩赛事**
   - 现象：部分联赛找不到
   - 原因：只读取竞彩开售的比赛
   - 影响：部分非主流联赛可能缺失

3. **RSSHub 不稳定**
   - 现象：知乎/微博资讯偶尔加载失败
   - 原因：公共实例限流/被封
   - 解决方案：配置多个备用源自动 fallback

### 8.2 功能相关

1. **Service Worker 休眠**
   - 现象：长时间不使用扩展后，数据可能延迟
   - 原因：Chrome 会自动挂起不活跃的 Service Worker
   - 解决：打开弹窗会立即刷新

2. **通知可能不出现**
   - 现象：系统通知不弹出
   - 可能原因：
     - 系统禁用了 Chrome 通知
     - 专注模式/勿扰模式开启
     - 通知被系统归类为不重要
   - 解决：检查系统通知设置

3. **语音播报无声**
   - 现象：语音播报没有声音
   - 可能原因：
     - 系统未安装中文语音包
     - 浏览器 TTS 权限问题
   - 解决：安装中文语音包

### 8.3 兼容性

1. **浏览器兼容性**
   - Chrome 88+：✅ 完全支持
   - Edge 88+：✅ 完全支持
   - Firefox：❌ 不支持（API 差异）
   - Safari：❌ 不支持

2. **操作系统兼容性**
   - Windows：✅ 完全支持
   - macOS：✅ 完全支持
   - Linux：✅ 支持（语音可能有问题）
   - ChromeOS：✅ 完全支持
   - Android/iOS：❌ 不支持

## 9. 常见问题

### Q1: 为什么看不到赛事数据？

**A**: 请检查：
1. 网络连接是否正常
2. 是否能访问 500.com
3. 点击弹窗的刷新按钮手动刷新
4. 等待 2 分钟自动刷新

### Q2: 为什么没有电竞实时比分？

**A**: 主流电竞数据网站（Liquipedia、HLTV、vlr.gg）都被 Cloudflare 反爬保护，无法直接爬取。目前使用静态赛程数据显示赛事时间和状态。

### Q3: 资讯加载失败怎么办？

**A**:
1. 检查网络连接
2. 尝试手动刷新
3. 在设置中切换其他资讯源
4. RSS 源有多个备用地址，会自动 fallback

### Q4: 如何添加新的赛事？

**A**:
1. 打开 `data/leagues.js`
2. 在对应分类下添加新赛事对象
3. 确保配置了正确的关键词用于匹配
4. 重新加载扩展

### Q5: 数据会占用多少存储空间？

**A**:
- 设置数据：约 5-10KB（sync storage）
- 赛事缓存：约 50-100KB（local storage）
- 资讯缓存：约 100-200KB（local storage）
- 历史记录：根据保留天数决定，默认 1 年约 1-2MB

### Q6: 会泄露我的隐私吗？

**A**: 不会。
- 所有数据都存储在本地浏览器中
- 不会上传任何个人数据
- 不会追踪用户行为
- 所有网络请求仅用于获取赛事和资讯数据

### Q7: 如何备份我的设置？

**A**:
1. 打开设置页面
2. 滚动到「数据管理」
3. 点击「导出数据」
4. 保存 JSON 文件

### Q8: 通知太多了怎么办？

**A**:
1. 提高「最低通知热度」到 S+ 级
2. 关闭不需要的通知类型
3. 只关注感兴趣的联赛
4. 关闭资讯浮层通知

### Q9: 为什么网页浮层不显示？

**A**:
1. 确认 content.js 在 manifest 中正确注册
2. 刷新网页
3. 检查是否有比分/资讯更新
4. 浮层只在有新更新时显示

### Q10: 热度分数是怎么计算的？

**A**: 热度分 = 基础关注度分 + 状态加成 + 比分加成。
- 基础分：根据赛事级别（S+=100, S=80, A=60, B=40, C=20）
- 状态加成：进行中+30，即将开始+15，已结束+10
- 比分加成：总得分越高加成越多，分差越小加成越多
- 最高上限：100分

### 10.2 添加新的资讯源

在 `data/sources.js` 中添加新源配置：

```javascript
new_source: {
  name: '源名称',
  icon: '📰',
  enabled: true,           // 默认是否开启
  url: 'https://example.com', // 主URL
  fallbackUrls: [],        // 备用URL
  method: 'GET',           // 请求方法
  type: 'html',            // html / rss / api
  parser: 'html_custom',   // 解析器名称
  interval: 15,            // 刷新间隔（分钟）
  filter: ['关键词'],      // 关键词过滤
  needsToken: false,       // 是否需要Token
},
```

在 `background.js` 的 `parseNews` 函数中添加解析器：

```javascript
function parseCustom(html, src) {
  const feeds = [];
  // 使用正则解析 HTML
  const itemRe = /<a[^>]*href="([^"]+)"[^>]*>([^<]+)<\/a>/gs;
  const matches = [...html.matchAll(itemRe)];
  
  for (let i = 0; i < Math.min(matches.length, 20); i++) {
    feeds.push({
      id: `custom-${i}-${Date.now()}`,
      title: matches[i][2].trim(),
      url: matches[i][1],
      source: src.name,
      sourceIcon: src.icon,
      hotScore: Math.round(80 - i * 3),
      time: new Date().toISOString(),
      type: 'news',
    });
  }
  return feeds;
}
```

在 `parseNews` 调度函数中添加 case。
